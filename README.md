# WebAR Story

A WebAR "story": point a phone browser at a printed marker and a video plays on it. Marker-based AR using [AR.js](https://github.com/AR-js-org/AR.js) + [A-Frame](https://aframe.io/) — runs entirely in the browser, no app install required.

This is a **multi-marker story**: one launch QR opens the page, then three separate markers each play their own video.

## The story flow

1. **Scan to open** — the standalone launch QR (`marker/qr-launch.png`) encodes the AR web page (<https://mrsdnf.github.io/WebAr/>). The user opens their phone camera and scans it to load the page (no app install).
2. **Aim at the markers** — once the page is open, the user aims the camera at any of three separate tracking markers:
   - `marker/marker-J.png` → plays video 1 (pattern `marker/J.patt`)
   - `marker/marker-S.png` → plays video 2 (pattern `marker/S.patt`)
   - `marker/marker-P.png` → plays video 3 (pattern `marker/P.patt`)

Each marker carries a single bold glyph (**J**, **S**, **P**) inside a solid black border, kept deliberately simple so it tracks reliably.

> **Videos:** all three markers currently use the placeholder clip `assets/clip.mp4`. Once the real videos are provided they will be wired in as `assets/clip1.mp4`, `assets/clip2.mp4`, and `assets/clip3.mp4` (one per marker), replacing the placeholder `src` values in `index.html`.

> The launch QR and markers are generated by `build_story_markers.py`. If the GitHub Pages URL ever changes, update `URL` in that script and regenerate so the QR points at the new address.

## Print the markers

Print each marker (`marker-J.png`, `marker-S.png`, `marker-P.png`) on white paper or card, sized so the marker square is roughly **5–8 cm**. Keep each square's solid **black border fully intact** with a white margin around it — do not crop it; the border is what the tracker locks onto. Print `marker/qr-launch.png` separately as the launch QR.

## Test locally (desktop webcam)

```bash
cd ~/projects/webar-card
python3 -m http.server 8080
```

Then open <http://localhost:8080> in a browser, allow camera access, and hold a marker (printed, or just shown on another screen) in front of the webcam. `localhost` is exempt from the HTTPS-camera rule, so this works without any certificate setup.

## Test on a phone (needs HTTPS)

Browsers only grant camera access over **HTTPS**, so a plain `http://<your-ip>:8080` will not work on a phone. Two options:

- **Tunnel it:** run `ngrok http 8080` and open the generated `https://…` URL on the phone.
- **Host it:** push this folder to a free static host (GitHub Pages / Netlify / Cloudflare Pages) and open that HTTPS link on the phone.

## Tips

- Use good, even lighting and hold each marker steady.
- If tracking is jittery, print the marker **larger** and keep the **black border clean** (no smudges, no glare, no fold across it).

## The videos

Each marker plays its video on a flat plane via an `<a-video>` element inside its `<a-marker>` in `index.html`.

On iOS the videos must stay `muted` and `playsinline`, and may need a screen tap to start (mobile autoplay restriction); `index.html` also kicks playback on `markerFound`.

## Use your own design on a marker (custom marker)

This project uses custom pattern markers (`marker/J.patt`, `marker/S.patt`, `marker/P.patt`, printed as the `marker-*.png` images). To put your own artwork on a card instead:

1. Open the AR.js Marker Training generator: <https://ar-js-org.github.io/AR.js/three.js/examples/marker-training/examples/generator.html>
2. Upload your image to generate a `.patt` file and save it under `marker/` (e.g. `marker/custom.patt`).
3. In `index.html`, point the marker tag at your pattern:

   ```html
   <a-marker type="pattern" url="marker/custom.patt">
   ```

Note that custom markers still need a solid black square border to track well. (The old `marker/hiro.png` is left in the repo but is no longer an active marker.)

## Future upgrade note

To track **arbitrary artwork or photos** (not a square fiducial marker), switch from AR.js to the [MindAR](https://github.com/hiukim/mind-ar-js) image-tracking library, which does natural-feature image tracking.
